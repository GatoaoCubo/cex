# Kubernetes AI Requirement (KAR) Conformance Artifact

The CNCF Kubernetes AI Requirement (KAR) defines conformance criteria for AI workloads in Kubernetes environments. Key requirements include:

- **GPU Topology Awareness**: Support for GPU resource discovery and scheduling
- **InfiniBand Integration**: Native support for high-speed interconnects
- **MIG (Multi-Instance GPU)**: Enable GPU partitioning for multi-tenant workloads
- **DRA (Device Resource Allocation)**: Dynamic resource allocation for accelerators
- **Checkpoint PVCs**: Persistent volume claims for model checkpointing
- **Multi-Node Training**: Distributed training frameworks with network optimization

This artifact provides the technical specifications for Kubernetes operators to ensure AI workloads meet KAR standards for performance, scalability, and resource efficiency.
