## Purpose

TypeScript interface contract for all Supabase tables in the commerce data layer — generated by `supabase gen types typescript` and extended with domain-level type aliases, providing type-safe DB access across edge functions and the React frontend.

## When to Use

- Importing typed Supabase client in edge functions or React components.
- Adding a new table: update this interface contract + regenerate types.
- Code review: verifying that DB access code matches the actual schema.
- Onboarding: understanding the data model at a glance.

## Typed Client Usage

```typescript
import { createClient } from '@supabase/supabase-js';
import type { Database } from './types';

const supabase = createClient<Database>(url, key);

// Fully typed: upsert product
const { error } = await supabase
  .from('products')
  .upsert({ sku: 'CAT-001', title: 'Ração Premium', price: 49.90, stock: 100, status: 'active' });

// Typed select
const { data: products } = await supabase
  .from('products')
  .select('id, sku, title, stock')
  .eq('status', 'active');
// products: Pick<Product, 'id' | 'sku' | 'title' | 'stock'>[]
```
