# Audit Summary

The 13-ISO family for `planning_strategy` generated by qwen3:8b fundamentally
**misreads the kind**. `planning_strategy` in CEX means the algorithmic
contract an LLM agent uses to decompose a goal (ReAct, Tree-of-Thoughts,
MCTS, HTN, Plan-and-Execute, Reflexion, Self-Refine, LLMCompiler). The weak
model instead produced artifacts about **business/corporate strategy**:
MoSCoW prioritization, KPIs, CTO emergency overrides, stakeholder engagement
plans, PMBOK standards, Urban Mobility Expansion Plans, Pandas/Backtrader,
and $500M budgets. Zero of the 13 ISOs mention ReAct, ToT, CoT, HTN, Reflexion,
or any concrete step schema (thought/action/observation). Pillar is also
mis-assigned as P03 (Prompt) when it should be P08 (Architecture).

## Critical Issues

1. **Domain inversion (all 13)**: corporate strategy instead of agent planning algorithms. Anti-example in bld_examples even flips the rule -- calls "Prompt-Based Decision Framework" an anti-pattern when *that is exactly what planning_strategy IS*.
2. **Wrong pillar (10 of 13)**: tagged P03 or mixed; should be P08 (Architecture) per task spec.
3. **ID regex wrong**: schema says `^p03_ps_[a-zA-Z0-9_-]+\.md$`; quality_gate says `P[0-9]{2}-[A-Z]{3}`. Conflict and both wrong (should be `^p08_ps_[a-z0-9_]+$`).
4. **Fabricated tools** (bld_tools): `cex_optimizer.py`, `cex_validator.py`, `strategy_checker.py`, `risk_analyzer.py`, `compliance_scanner.py`, `performance_simulator.py`, `Backtrader`, `Pandas` -- none exist in CEX and none are relevant to agent planning.
5. **No step schema anywhere**: zero mention of thought/action/observation/reflection structure.
6. **No named algorithms**: ReAct, ToT, MCTS, HTN, Plan-and-Execute, Reflexion, Self-Refine, LLMCompiler, CoT -- none cited.
7. **No branching params**: max_depth, max_breadth, selection policy, expansion strategy -- absent.
8. **No termination criteria**: goal_reached / budget_exhausted / confidence_drop / no_progress -- absent.
9. **No tradeoff analysis**: ReAct's brittleness, ToT's cost, HTN's rigidity -- never discussed.
10. **quality: null violated** in knowledge_card_builder gold reference comparison (KC builder has `quality: 9.1` -- gold), but weak-model outputs all have `quality: null` which is correct-by-accident.

## Rewrites Applied

Three lowest-scoring files rewritten in-place (preserving naming). All rewrites 50-80 lines, complete frontmatter, P08 pillar, concrete agent-planning content.

### 1. `bld_manifest_planning_strategy.md` (was 3.7 -> target 9.0+)
- Fixed pillar P03 -> P08
- Identity now references Yao 2022 (ReAct), Shinn 2023 (Reflexion)
- Capabilities list concrete parameters (max_depth, max_branches, temperature, confidence_threshold)
- Routing keywords are real algorithm names
- Crew Role disambiguates against workflow/chain/agent/tool_router

### 2. `bld_output_template_planning_strategy.md` (was 2.1 -> target 9.0+)
- Removed all `{{placeholder}}` stubs
- Added concrete Step Schema YAML (thought/action/action_input/observation/reflection/confidence/parent_step_id)
- Branching Policy table (max_depth, max_breadth, selection, expansion) with per-algorithm defaults
- Budget & Termination table (6 criteria)
- Revision / Reflection block
- Tradeoffs section (strengths / weaknesses / avoid-when)
- Example Trace skeleton

### 3. `bld_schema_planning_strategy.md` (was 4.2 -> target 9.0+)
- ID pattern fixed: `^p08_ps_[a-z0-9_]+$`
- Added required fields: `strategy_class` enum, `algorithm` enum
- Class-to-algorithm validity mapping (linear->{react,cot}, tree->{tot,mcts}, etc.)
- Body structure is 7 ordered sections (Summary, Step Schema, Branching, Budget, Revision, Tradeoffs, Trace)
- Cross-field constraints: reflexion/self_refine REQUIRE Revision.mechanism != none; tree class REQUIRES max_breadth >= 2
