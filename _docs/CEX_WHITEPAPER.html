1. The Problem

When you ask an LLM to build an agent, each model generates a different structure. Fields are invented ad-hoc. Formats are incompatible. No shared vocabulary, no schema, no quality contract.

CEX defines a universal typed dictionary: 99 artifact types (kinds), each with a schema, a builder, and quality gates. Any LLM produces structurally compatible artifacts.
CEX is NOTCEX IS
An agent framework (LangChain, CrewAI)A construction dictionary any framework consumes
A prompt libraryA typed taxonomy (prompts = 1 of 99 kinds)
Tied to one LLMMulti-LLM (Claude, GPT, Gemini, Ollama)
A runtime engineA build-time system producing validated artifacts

2. CEX in One Page

L1: DICTIONARY — 99 KINDS in 12 PILLARS Each kind: schema + builder (13 ISOs) + KC + examples Built via 8F PIPELINE (8 sequential functions) L2: NUCLEI — 7 composed units (N01-N07) Each uses 12-25 kinds from L1 N03 = Builder N07 = Orchestrator L3: INSTANCES — concrete project fills _instances/your-project/N01-N07/

Numbers: 99 kinds, 12 pillars, 8 functions, 7 nuclei, 1,287 builder ISOs, 99 KCs, 17 tools.

3. The 99 Kinds

A kind is a named artifact type with: definition, boundary (what it is NOT), pillar, max size, naming convention, and 8F function mapping. Source of truth: .cex/kinds_meta.json.

PillarName#8F FnExample Kinds

P02Model11BECOMEagent, axiom, mental_model, router

P05Output4GOVERNresponse_format, formatter, parser

P07Evals10GOVERNbenchmark, scoring_rubric, quality_gate
P08Architecture8INJECTagent_card, pattern, diagram

P11Feedback6CONSTRAINquality_gate, guardrail, bugloop
P12Orchestration8PRODUCEworkflow, dag, dispatch_rule, handoff

Boundary Discipline

Every kind declares what it is NOT: agent = "NOT a skill, NOT a tool, NOT a prompt". workflow = "NOT a chain, NOT a DAG". This prevents scope creep.

4. The 12 Pillars

Pillars group related kinds and map to 8F functions:

P01 Knowledge -----[INJECT]------> F3 (load knowledge) P02 Model ---------[BECOME]------> F2 (load identity) P03 Prompt --------[INJECT]------> F3 (load prompts) P04 Tools ---------[CALL]--------> F5 (load tools) P05 Output --------[GOVERN]------> F7 (validate output) P06 Schema --------[CONSTRAIN]---> F1 (load constraints) P07 Evals ---------[GOVERN]------> F7 (validate quality) P08 Architecture --[INJECT]------> F3 (load patterns) P09 Config --------[GOVERN]------> F1 (load config) P10 Memory --------[INJECT]------> F3 (load memory) P11 Feedback ------[CONSTRAIN]---> F7 (load guardrails) P12 Orchestration -[PRODUCE]-----> F6/F8 (build flows)

Axiom AX03: One kind, one pillar. If a knowledge_card starts explaining a workflow, STOP. Separate artifacts.

5. The 8F Pipeline

Every artifact is produced by 8 sequential functions:

#FunctionVerbInputOutputPillars
F1CONSTRAINRestrictKind nameSchema + configP06, P09
F2BECOMEBeKind nameBuilder identity (13 ISOs)P02
F3INJECTKnowKind + contextKCs + examples + memoryP01,P03,P08,P10
F4REASONThinkF1+F2+F3Construction planLLM
F5CALLDoTools listTool manifestP04
F6PRODUCEGeneratePlan + toolsArtifact draftP12
F7GOVERNEvaluateDraft + schemaPass/Fail + scoreP05,P07,P11
F8COLLABORATECoordinateValidated artifactSaved+compiled+indexedP12

F6→F7 Retry Loop

F6 PRODUCE -----> F7 GOVERN ^ | | PASS ---------+-------> F8 COLLABORATE (save) | | +-- FAIL ---------+ (max 2 retries, then REJECT)

Example

Intent: "quality gate for API responses"
[MOTOR] resolves: kind=quality_gate, pillar=P07
[RUNNER] F1: Load schema  F2: Load builder ISOs
         F3: Load KC      F4: Plan approach
         F5: Scan similar F6: Generate artifact
         F7: Validate     F8: Save + compile + signal
Output: quality_gate_api_response.md (score: 8.7)

Total: 99 kinds × 13 ISOs = 1,287 instruction files

Builders are the DNA of CEX. When an LLM loads a builder, it acquires specialized knowledge to construct that kind — regardless of which LLM executes.

Layer 2: 12LP Checklist

12 checkpoints: frontmatter, kind match, pillar alignment, schema, density, completeness, uniqueness, references, boundary, size, naming, "stranger test."

Layer 3: 5D Scoring

DimensionWeightMeasures
D1: Structural Integrity25%Frontmatter, schema, naming
D2: Content Density20%Signal-to-noise ratio
D3: Actionability25%Usable by stranger in 2min?
D4: Boundary Discipline15%Single kind, single responsibility
D5: Composability15%Clean interfaces, crew-compatible

Final = (D1×0.25)+(D2×0.20)+(D3×0.25)+(D4×0.15)+(D5×0.15)

11. The Strange Loop

N03 was built using the 8F pipeline it defines. Self-referential: the factory IS the product.

3-Phase Bootstrap

PhaseNameWhat
1GENESIS7 core artifacts written manually
2AUTOPOIESISN03 builds remaining 19 via 8F
3REWRITEN03 rewrites originals with full context
Intentional Strange Loop (Hofstadter, 1979). Loop closes when Phase 3 outputs are strictly better than Phase 1 originals.

10 Axioms

AXStatement
01Every artifact has YAML frontmatter. No exceptions.
02Every kind has exactly one builder with 13 ISOs.
03One kind, one pillar. Boundary is law.
04The 8F pipeline is the only construction method.
05Builders are instructions, not executors. Any LLM runs them.
06Quality floor is 8.0. Below = not published.
07Template-First: search before creating.
08Compilation (.md → .yaml) is mandatory post-save.
09The index is discovery. Re-index after every save.
10Open variables ({{mustache}}) are deliberate.

12. Quick Start

git clone https://github.com/GatoaoCubo/cex.git
cd cex && pip install pyyaml

Boot

cex              # N07 orchestrator (pi + opus)
cex doctor       # Health check
cex status       # Monitor builders
cex stop         # Kill all

Build an Artifact

python _tools/cex_8f_motor.py --intent "quality gate for API"
python _tools/cex_8f_runner.py --kind quality_gate
python _tools/cex_doctor.py
python _tools/cex_compile.py P07_evals/quality_gate_api.md

Parallel Grid

powershell -File _spawn/spawn_solo.ps1 -nucleus n03 -task "build" -interactive
powershell -File _spawn/spawn_grid.ps1 -mission WAVE1 -interactive
powershell -File _spawn/spawn_monitor.ps1
