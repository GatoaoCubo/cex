## 1.1 The 8 LLM Functions (Pipeline Layer)

The 12 pillars organize ARTIFACTS (filesystem). The 8 functions describe what the LLM DOES (pipeline).
Every LLM system -- from prompt to agent group -- executes these functions in this order:

| # | Function | Verb | What It Does | Pillars |
|---|----------|------|-------------|---------|
| 1 | CONSTRAIN | Restrict | Kind resolution, schema loading | P06, P08 |
| 2 | BECOME | Be | Identity, persona, builder loading | P02, P03 |
| 3 | INJECT | Know | Context, knowledge, memory, brand | P01, P10 |
| 4 | REASON | Think | CoT, planning, GDP gate | P03 |
| 5 | CALL | Do | Tools, APIs, MCPs, retrieval | P04 |
| 6 | PRODUCE | Generate | Final output artifact | P03, P05 |
| 7 | GOVERN | Evaluate | Quality gates, evals, scoring | P07, P09, P11 |
| 8 | COLLABORATE | Coordinate | Signals, handoffs, dispatch | P12 |

**Law**: functions are PIPELINE (sequence), not categories (folders).

## 4. Dual Format (Mandatory)

| What | Source (.md) | Compiled | Sync Tool |
|------|-------------|----------|-----------|
| Artifacts (123 kinds) | YAML frontmatter + MD body | `.yaml` or `.json` | cex_compile.py |
| Schemas | `.yaml` only | N/A | manual |
| Builders (13 ISOs) | `.md` only | N/A | manual |
| Meta-docs | `.md` only | N/A | manual |

**Rule**: every artifact instance exists in TWO formats. The .md is the authoring source. The compiled/ is the consumption output.

## 5. Schema Inheritance

```
P{NN}_*/_schema.yaml              ROOT (source of truth)
  |
  +-- N{XX}/P{NN}/_schema.yaml    NUCLEUS (inherits + specializes)
       inherits: root
       fields_add: {}              Extra fields for this nucleus
       constraints_override: {}    Specific constraints
```

**Rule**: a nucleus NEVER removes fields from root. It only ADDS or RESTRICTS.

## 7. Quality Gates

| Score | Tier | Destination |
|-------|------|------------|
| >= 9.5 | Golden | Reference pool |
| >= 8.0 | Skilled | Published |
| >= 7.0 | Learning | Experimental |
| < 7.0 | Rejected | Rebuild |

Minimum density: 0.85. Max size: per _schema.yaml. Scoring: 3-layer hybrid (structural 30% + rubric 30% + semantic 40%).

4. NEVER remove field from schema (deprecate only)

6. NEVER create kind outside P{NN}/_schema.yaml root
7. ALWAYS dual output for artifacts
8. ALWAYS density >= 0.85
9. ALWAYS builder builds, human reviews

## 10. Runtime Architecture

```
User --> boot/cex.ps1 --> claude (Opus 4.6, 1M context)
                             |
                             +--> CLAUDE.md (system instructions)
                             +--> .claude/rules/ (nucleus rules)
                             +--> .claude/agents/ (125 sub-agents)
                             +--> .claude/commands/ (slash commands)
                             |
                             +--> N07 Orchestrator
                                    |
                                    +--> bash _spawn/dispatch.sh grid MISSION
                                    |       |
                                    |       +--> boot/n01.ps1 (claude, opus-4-6)
                                    |       +--> boot/n02.ps1 (claude, opus-4-6)
                                    |       +--> boot/n03.ps1 (claude, opus-4-6)
                                    |       +--> boot/n04.ps1 (claude, opus-4-6)
                                    |       +--> boot/n05.ps1 (claude, opus-4-6)
                                    |       +--> boot/n06.ps1 (claude, opus-4-6)
                                    |
                                    +--> .cex/runtime/ (handoffs, signals, pids)
                                    +--> _tools/ (91 Python scripts)
```

**Dependencies**: Python 3.12+, Node.js 18+, Git 2.40+, Claude Code CLI.
**Auth**: Anthropic account (Max/Pro/API key). First run opens browser for OAuth.
**Install**: `setup.cmd` or `npm install -g @anthropic-ai/claude-code` + `pip install pyyaml tiktoken`.
