## Problem

After STRUCT_ALIGN V1, there is a structural duality:

```
cex/
  P01_knowledge/       root level -- _schema, library, examples, templates
  N00_genesis/
    P01_knowledge/     N00 level -- kind manifests, kind indexes (W4)
  N01_intelligence/
    P01_knowledge/     nucleus level -- N01 artifacts
```

Three layers where two would suffice. N00 IS the pillar registry (archetype).
The root P01-P12 dirs are a historical artifact, not intentional design.

**Consequence**: LLM navigating the repo finds pillar data in two places.
`P01_knowledge/library/kind/kc_*.md` should live at `N00_genesis/P01_knowledge/library/kind/kc_*.md`.

## Vision

```
cex/
  N00_genesis/
    P01_knowledge/        MERGED: _schema + library + examples + templates + kind manifests
    P02_model/            MERGED: _schema + examples + templates + kind manifests
    ...
    P12_orchestration/    MERGED: _schema + examples + templates + dags + workflows + manifests
  N01-N07/
    P01_knowledge/        nucleus artifacts (unchanged -- already P-numbered)
    ...
  archetypes/             unchanged
  _tools/                 unchanged
```

Root P01-P12 dirs are DELETED after merge. N00 is the single pillar registry.

### Wave 1: Merge Tool (1 file -- N05)

| Action | Path | Kind | Est. Size | Notes |

| CREATE | `_tools/cex_merge_pillars.py` | cli_tool | 6KB | Moves root P{xx}_* content into N00_genesis/P{xx}_*; validates + executes |

**Interface:**
```
python _tools/cex_merge_pillars.py --dry-run      # list all operations
python _tools/cex_merge_pillars.py --execute      # run git mv + git rm
python _tools/cex_merge_pillars.py --verify       # check N00 has all content
```

**Logic:**
1. For each pillar P01-P12:
   - List all items in root `P{xx}_*/` (except `compiled/` -- skip)
   - For each item: `git mv P{xx}_{name}/{item} N00_genesis/P{xx}_{name}/{item}`
   - If item already exists in N00 target: print `[CONFLICT] {item}` and skip
2. After all moves: `git rm -r P{xx}_*/` for each root pillar
3. Verify: `ls N00_genesis/P01_knowledge/library/` should exist
