# Spec N06 -- Brand Verticalization: The Missing Variable

## THE PROBLEM

CEX is a brain without a face. 7 nuclei, 12 pillars, 108 builders, 8F pipeline -- all generic.
N02 writes copy but doesn't know WHOSE voice. N05 deploys but doesn't know WHOSE product.
N01 researches but doesn't know WHOSE market. N03 builds but doesn't know WHOSE palette.

The missing X is `{{BRAND}}` -- a mustache variable layer that, once filled, transforms
a generic CEX instance into a specialized agent ecosystem for ONE specific brand/company.

N06 is the nucleus that CREATES and PROPAGATES this identity.

## THE VISION

N06 evolves from "Commercial & Monetization" into **Brand Architect** -- the nucleus that:

1. **Discovers** brand identity from user input (brand discovery interview)
2. **Generates** the Brand Book (the canonical document)
3. **Extracts** mustache variables into `brand_config.yaml` (machine-readable)
4. **Propagates** brand context to ALL other nuclei via injection
5. **Monetizes** the brand (pricing, funnels, courses -- original scope KEPT)

N06 = Brand Identity + Brand Monetization. Discovery first, then revenue.

## THE MUSTACHE VARIABLE SYSTEM

### brand_config.yaml (Single Source of Truth)

Every CEX instance gets ONE file: `.cex/brand/brand_config.yaml`
N06 generates it. All nuclei read it. No hardcoded brand anywhere.

```yaml
# .cex/brand/brand_config.yaml
# Generated by N06 Brand Architect after Brand Discovery
# This file makes your CEX instance YOURS

identity:
  BRAND_NAME: "{{BRAND_NAME}}"           # Company/product name
  BRAND_TAGLINE: "{{BRAND_TAGLINE}}"     # One-line hook
  BRAND_SLOGAN: "{{BRAND_SLOGAN}}"       # Campaign slogan (optional)
  BRAND_MISSION: "{{BRAND_MISSION}}"     # Why we exist
  BRAND_VISION: "{{BRAND_VISION}}"       # Where we're going
  BRAND_VALUES: [{{BRAND_VALUES}}]       # 3-5 core values
  BRAND_STORY: "{{BRAND_STORY}}"         # Origin narrative (2-3 paragraphs)

archetype:
  BRAND_ARCHETYPE: "{{BRAND_ARCHETYPE}}" # 1 of 12 Jungian (creator, hero, sage...)
  BRAND_ARCHETYPE_SHADOW: "{{SHADOW}}"   # What to AVOID (counter-archetype)
  BRAND_PERSONALITY: [{{TRAITS}}]        # 5 personality traits

voice:
  BRAND_VOICE_TONE: "{{TONE}}"           # e.g. "tecnico-confiante, direto"
  BRAND_VOICE_FORMALITY: {{1-5}}         # 1=casual 5=formal
  BRAND_VOICE_ENTHUSIASM: {{1-5}}
  BRAND_VOICE_HUMOR: {{1-5}}
  BRAND_VOICE_WARMTH: {{1-5}}
  BRAND_VOICE_AUTHORITY: {{1-5}}
  BRAND_VOICE_DO: [{{DO_LIST}}]          # "Use X, prefer Y"
  BRAND_VOICE_DONT: [{{DONT_LIST}}]      # "Never X, avoid Y"
  BRAND_LANGUAGE: "{{LANG}}"             # pt-BR, en-US, etc.

audience:
  BRAND_ICP: "{{ICP_DESCRIPTION}}"       # Ideal Customer Profile (2-3 sentences)
  BRAND_ICP_AGE: "{{AGE_RANGE}}"
  BRAND_ICP_LOCATION: "{{LOCATION}}"
  BRAND_ICP_INCOME: "{{INCOME_CLASS}}"
  BRAND_ICP_VALUES: [{{ICP_VALUES}}]
  BRAND_ICP_FEARS: [{{ICP_FEARS}}]
  BRAND_ICP_ASPIRATIONS: [{{ICP_ASPIRATIONS}}]
  BRAND_TRANSFORMATION: "From {{BEFORE}} to {{AFTER}} through {{BRIDGE}}"

visual:
  BRAND_COLORS:
    primary: "{{PRIMARY_HEX}}"
    secondary: "{{SECONDARY_HEX}}"
    accent: "{{ACCENT_HEX}}"
    background: "{{BG_HEX}}"
    foreground: "{{FG_HEX}}"
    surface: "{{SURFACE_HEX}}"
  BRAND_FONTS:
    heading: "{{HEADING_FONT}}"
    body: "{{BODY_FONT}}"
    mono: "{{MONO_FONT}}"
  BRAND_LOGO_URL: "{{LOGO_URL}}"
  BRAND_FAVICON_URL: "{{FAVICON_URL}}"
  BRAND_STYLE: "{{STYLE}}"              # "minimal-dark", "vibrant-light", etc.

positioning:
  BRAND_CATEGORY: "{{CATEGORY}}"
  BRAND_UVP: "Para {{TARGET}} que {{NEED}}, {{BRAND}} e {{CATEGORY}} que {{BENEFIT}} porque {{REASON}}"
  BRAND_DIFFERENTIATOR: "We are the only {{CATEGORY}} that {{UNIQUE}}"
  BRAND_COMPETITORS: [{{COMPETITOR_LIST}}]
  BRAND_CONTENT_PILLARS: [{{PILLAR_LIST}}]  # 3-5 content themes

monetization:
  BRAND_PRICING_MODEL: "{{MODEL}}"       # subscription, one-time, credit-based, freemium
  BRAND_CURRENCY: "{{CURRENCY}}"         # BRL, USD, EUR
  BRAND_PRICE_ANCHOR: "{{ANCHOR}}"       # Highest-tier price (anchoring)
  BRAND_TIERS: [{{TIER_LIST}}]           # free, starter, pro, enterprise
  BRAND_PAYMENT_PROVIDERS: [{{PROVIDERS}}] # stripe, mercadopago, hotmart
```

### How Brand Propagates to Nuclei

```
N06 (generates brand_config.yaml)
  |
  +---> N01 reads BRAND_ICP, BRAND_COMPETITORS, BRAND_CATEGORY
  |       -> Research is targeted to the right market
  |
  +---> N02 reads BRAND_VOICE, BRAND_COLORS, BRAND_FONTS, BRAND_VALUES
  |       -> Copy matches brand tone, HTML uses brand palette
  |
  +---> N03 reads BRAND_COLORS, BRAND_FONTS, BRAND_STYLE
  |       -> Components use brand design tokens
  |
  +---> N04 reads BRAND_NAME, BRAND_CATEGORY, BRAND_CONTENT_PILLARS
  |       -> Knowledge indexed under brand context
  |
  +---> N05 reads BRAND_NAME, BRAND_LOGO_URL
  |       -> Deploys with correct branding, env vars
  |
  +---> N07 reads BRAND_NAME for commit messages, logging
  |
  +---> boot/*.cmd reads BRAND_NAME for window titles
  +---> CLAUDE.md reads BRAND_NAME, BRAND_TAGLINE for system identity
```

### Injection Mechanism

```python
# _tools/brand_inject.py (new tool)
# Reads .cex/brand/brand_config.yaml
# Replaces {{BRAND_*}} in any template
# Called by crew_runner before prompt composition

def inject_brand(template: str, brand_config: dict) -> str:
    for key, value in flatten(brand_config).items():
        template = template.replace(f"{{{{{key}}}}}", str(value))
    return template
```

## 8F DECOMPOSITION (CONSTRAIN > BECOME > INJECT > REASON > CALL > PRODUCE > GOVERN > COLLABORATE)

### F1 CONSTRAIN (Schemas)
- N06 output MUST produce valid `brand_config.yaml` (schema-validated)
- Brand Book MUST cover: identity, archetype, voice, audience, visual, positioning
- Monetization artifacts MUST reference brand_config (never hardcode)
- All 12 archetypes available (Jungian), each with tone_of_voice scores

### F2 BECOME (Agent Identity)
- Agent: Brand Architect + Revenue Engineer (dual-role, like N02)
- Primary: Brand Discovery → Brand Book → brand_config.yaml → Propagation
- Secondary: Pricing → Funnels → Courses → Revenue (original N06 scope)
- Model: sonnet (discovery/copy) | opus (complex brand systems)

### F3 INJECT (Knowledge)
- **REUSE**: 12 Jungian archetypes (from legacy marca package)
- **REUSE**: Ries & Trout positioning framework
- **REUSE**: Brand Discovery prompts (3-phase: discovery → archetype → positioning)
- **NEW KCs needed**: brand_book_patterns, brand_voice_systems, ICP_frameworks,
  brand_propagation_architecture, design_token_brand_mapping,
  competitive_positioning_methods, brand_naming_patterns

### F4 REASON (Prompts)
- Brand Discovery Interview: 12-15 questions, progressive depth
- Brand Book Generator: from answers -> structured document
- brand_config.yaml Extractor: from Brand Book -> machine-readable YAML
- Brand Audit: score existing brand against archetype alignment

### F5 CALL (Tools)
- `brand_inject.py`: template BRAND_* variable replacement
- `brand_validate.py`: validate brand_config.yaml against schema
- `brand_propagate.py`: push brand context to all nuclei prompts
- `brand_audit.py`: score brand consistency across artifacts
- KEEP: pricing_calculator.py, funnel_mapper.py, revenue_forecaster.py

### F6 PRODUCE (Output Templates)
- Brand Book (40-60 page equivalent MD document)
- brand_config.yaml (the mustache variable file)
- Brand One-Pager (executive summary, 1 page)
- Brand Voice Guide (tone matrix + examples per channel)
- Visual Identity Sheet (colors + fonts + logo usage)
- Competitive Positioning Map (2D grid + whitespace)
- Transformation Arc (before -> bridge -> after)
- KEEP: pricing table, funnel map, course outline, revenue model

### F7 GOVERN (Quality Gates)
- Brand coherence: archetype alignment across all outputs
- Voice consistency: tone scores match across copy samples
- Config completeness: all REQUIRED fields in brand_config.yaml filled
- Propagation test: every nucleus can resolve its BRAND_* variables
- Visual harmony: color contrast ratios meet WCAG, font pairing validated

### F8 COLLABORATE (Orchestration)
- N06 is the FIRST nucleus to run on new CEX instance (brand before everything)
- After brand_config.yaml exists, other nuclei can resolve variables
- N06 handoff to N02: "Brand Book done, use BRAND_VOICE for all copy"
- N06 handoff to N03: "Design tokens extracted, use BRAND_COLORS/FONTS"
- N06 handoff to N05: "Deploy config ready, BRAND_NAME for service labels"
