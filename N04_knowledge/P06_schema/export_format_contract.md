# Export Format Contract

## Format Specs

### YAML (CEX internal)
```yaml
# Generated by cex_compile.py
id: p01_kc_example
kind: knowledge_card
title: "Example KC"
body: "..."
```

### JSONL (Fine-Tuning)
```json
{"instruction": "Explain chain-of-thought prompting", "input": "", "output": "CoT elicits intermediate reasoning steps..."}
```
Fields: instruction (from tldr), input (from keywords), output (from body).

### SQL (Supabase)
```sql
INSERT INTO kcs (id, kind, pillar, domain, title, tldr, body, embedding, created, updated)
VALUES ('p01_kc_example', 'knowledge_card', 'P01', 'domain', 'Title', 'tldr', 'body', '[vector]', now(), now());
```

### CSV (ML Datasets)
```csv
id,kind,pillar,domain,title,tags,density,quality,word_count,created
p01_kc_example,knowledge_card,P01,domain,Title,"tag1;tag2",0.92,null,350,2026-03-31
```

## Usage Guidelines

| Format | When to Use | Implementation |
|--------|-------------|----------------|
| **YAML** | Internal CEX compilation, artifact storage | `cex_compile.py --all` |
| **JSONL** | LLM fine-tuning datasets, prompt-response pairs | `cex_export.py --format=jsonl --filter=kind:kc` |
| **SQL** | Supabase sync, vector search, production DB | `cex_export.py --format=sql --table=kcs` |
| **CSV** | ML analysis, Excel reports, data science | `cex_export.py --format=csv --fields=metadata` |

**Anti-patterns:**
- Using YAML for external integrations (not portable)
- JSONL without instruction quality validation
- SQL exports without proper escaping
- CSV with unescaped commas in content fields

## Validation

| Format | Test Command | Expected Output |
|--------|--------------|-----------------|
| **YAML** | `python _tools/cex_compile.py --validate P01/` | "✅ 12 artifacts compiled" |
| **JSONL** | `jq -c . export.jsonl \| head -1` | Valid JSON with instruction/input/output |
| **SQL** | `psql -d test -f export.sql` | No syntax errors, rows inserted |
| **CSV** | `python -c "import pandas as pd; pd.read_csv('export.csv')"` | No parsing errors |
