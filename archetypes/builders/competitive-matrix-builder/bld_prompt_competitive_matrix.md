## Phase 1: RESEARCH  
1. Identify target competitors and their product/service offerings.  
2. Gather feature data from public specs, customer reviews, and sales demos.  
3. Map customer requirements for sales battle cards and procurement evals.  
4. Analyze pricing models, SLAs, and integration capabilities.  
5. Document unique value propositions and differentiators.  
6. Validate data sources for accuracy and relevance.  

## Phase 2: COMPOSE  
1. Define matrix structure: rows=features, cols=competitors + self.  
2. Populate rows with prioritized features (e.g., scalability, security).  
3. Fill columns with competitive benchmarks (e.g., yes/no, rating scales).  
4. Align data with SCHEMA.md (e.g., feature categories, metadata fields).  
5. Use OUTPUT_TEMPLATE.md to format headers, footers, and annotations.  
6. Insert value propositions and differentiators in dedicated columns.  
7. Add procurement-specific metrics (e.g., TCO, implementation time).  
8. Cross-reference with sales battle card templates for consistency.  
9. Finalize with clear legends and notes for evaluators.  

## Phase 3: VALIDATE  
- [ ] ✅ All features mapped to customer requirements (Phase 1, Step 3).  
- [ ] ✅ Data sources cited and verified for accuracy (Phase 1, Step 6).  
- [ ] ✅ Schema alignment confirmed (Phase 2, Step 4).  
- [ ] ✅ Template formatting matches OUTPUT_TEMPLATE.md (Phase 2, Step 5).  
- [ ] ✅ Stakeholders confirm usability for sales/procurement scenarios.
