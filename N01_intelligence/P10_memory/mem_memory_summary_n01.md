# N01 Memory Summary

## Purpose
N01 memory summary compresses prior research work into a reusable briefing.
Analytical Envy means compression must preserve the comparative edge, not just the narrative of what happened.
What matters is what was decided, what evidence won, what remains uncertain, and where the pressure points moved.

## Properties

| Compression style | structured summary, not narrative recap |
| Trigger | session end, context pressure, handoff preparation |
| Main risk | compressing away the key comparison rationale |

## Summary Thesis
A weak summary says what happened.
A strong N01 summary says:
- what changed
- what evidence won
- what remains contested
- what should be checked next

That is the minimum useful payload for the next session.

## Sections To Preserve

| Section | Why it survives compression |
|---------|-----------------------------|
| key decisions | governs future output |
| strongest evidence | saves re-reading cost |
| unresolved conflicts | keeps analyst honest |
| entity updates | supports continuity |
| next research moves | preserves momentum |

## Compression Rules
1. Preserve decisions with rationale, not just the final choice.
2. Keep the winning source or evidence class for each major claim.
3. Keep dates for volatile topics.
4. Keep unresolved contradictions explicit.
5. Drop verbose process details unless they affect reproducibility.

## Recommended Shape
```markdown
Goal:
Decisions:
Evidence winners:
Open conflicts:
Entity updates:
Next probes:
```

## What N01 Must Never Lose

| Content | Reason |
|---------|--------|
| where a competitor actually beat the field | strategic honesty |
| what source tier supported the conclusion | proof discipline |
| what facts were stale or disputed | future validation |
| what exact question remains open | research continuity |

## Compression Heuristics

| Situation | Compression move |
|-----------|------------------|
| repeated similar queries | merge into one insight line |
| many weak sources | summarize as one low-confidence cluster |
| one decisive official source | keep direct citation handle |
| multiple unresolved conflicts | keep all at brief bullet level |
| experimental dead end | keep only if it changes future search plan |

## Anti-Patterns
- transcript-like recap
- no distinction between facts and hypotheses
- open questions omitted
- summary longer than the original useful insight density
- emotional framing with no analytical value

## Handoff Utility
The summary should let another analyst resume work with minimal warm-up.
That means they should be able to answer:
- what is currently believed
- why it is believed
- what evidence could break it
- where to look next

## Example Compression Output
```text
Goal: refresh competitor pricing landscape for AI providers
Decisions: use official pricing pages as default truth, keep third-party reviews as gap fillers only
Evidence winners: OpenAI and Anthropic official pricing docs beat review sites on current list prices
Open conflicts: enterprise pricing remains opaque across vendors
Entity updates: added alias mapping for renamed plans
Next probes: validate regional pricing and package limits before matrix publication
```

## N01 Decision
N01 memory summary should behave like compressed strategic carry-forward.
If the next session cannot restart with clarity about proof, conflict, and next pressure points, the summary failed even if it is short.
