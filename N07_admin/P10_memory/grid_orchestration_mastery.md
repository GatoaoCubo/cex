```powershell

using System;
using System.Runtime.InteropServices;
public class Win32Move {
    [DllImport("user32.dll")]
    public static extern bool MoveWindow(IntPtr hWnd, int X, int Y, int W, int H, bool r);

'@

[Win32Move]::MoveWindow($proc.MainWindowHandle, {x}, {y}, 640, 520, $true)
```

## 3. Crash Detection & Recovery

### Detect dead nucleus
```bash
# Check if LLM process exists under CMD
wmic process where "ParentProcessId={CMD_PID}" get ProcessId,Name
# If only conhost.exe → LLM crashed, CMD is zombie
# If node.exe/claude.exe/codex.exe present → alive
```

### Bulk health check
```bash
tasklist | grep -i "claude\|gemini\|codex\|node"
```

### Recovery procedure
```powershell
# 1. Kill zombie CMD
taskkill /PID {old_pid} /F

# 2. Ensure handoff is in place
cp .cex/runtime/handoffs/{MISSION}_{nucleus}.md .cex/runtime/handoffs/{nucleus}_task.md

# 3. Relaunch (use fallback if primary crashed)
$proc = Start-Process cmd -ArgumentList '/k "boot\n0X.cmd"' -WorkingDirectory $root -PassThru

# 4. Wait + reposition to grid slot
Start-Sleep -Seconds 3
[Win32Move]::MoveWindow($proc.MainWindowHandle, {x}, {y}, 640, 520, $true)

# 5. Update PID tracking
"{new_pid} n0x claude {session_id} {timestamp}" | Add-Content .cex/runtime/pids/spawn_pids.txt
```

## 4. Known Problems

### P1: Gemini CLI crash — Node.js v24 + https-proxy-agent
- **Symptom**: `node:internal/modules/cjs/loader` error on boot
- **Cause**: Node.js v24 incompatible with `https-proxy-agent` in gemini-cli
- **Intermittent**: N01 may boot fine while N04 crashes (same CLI, same Node)
- **Fix now**: Use `boot/n0X-claude.cmd` fallback
- **Fix permanent**: `npm update -g @google/gemini-cli` or downgrade Node to v22 LTS

### P2: Zombie CMD windows
- **Symptom**: CMD open but idle, no LLM child process
- **Detect**: `wmic process where "ParentProcessId={PID}" get ProcessId,Name`
- **Fix**: Kill + relaunch + reposition

### P3: PID file noise
- **Symptom**: `dispatch.sh status` shows CRASHED from old sessions
- **Fix pre-grid**: `> .cex/runtime/pids/spawn_pids.txt` (truncate)

### P4: Handoff not copied to n0X_task.md
- **Symptom**: Nucleus boots but has no task
- **Cause**: spawn_grid.ps1 copies `{MISSION}_{nucleus}.md` → `{nucleus}_task.md`
- **Fix manual**: `cp .cex/runtime/handoffs/{MISSION}_n0X.md .cex/runtime/handoffs/n0X_task.md`

### P5: Gemini nuclei don't git commit
- **Already in**: workflow_admin.md consolidate step
- **Reminder**: N07 MUST `git add N01_*/ N04_*/` after Gemini work completes

## 5. Diagnostic Commands

```bash
# Who's alive?
tasklist | grep -i "claude\|gemini\|codex\|node"

# What's under a specific CMD?
wmic process where "ParentProcessId={PID}" get ProcessId,Name,CommandLine

# Any signals yet?
ls .cex/runtime/signals/

# Recent commits from nuclei?
git log --oneline -10

# Full system health
python _tools/cex_doctor.py

# Grid status (dispatch wrapper)
bash _spawn/dispatch.sh status
```
