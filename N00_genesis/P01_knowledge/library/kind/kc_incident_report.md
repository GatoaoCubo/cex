# Incident Report Template

## Purpose
Document AI system failures, security breaches, or unexpected behavior with structured analysis

## Key Sections
1. **Summary**  
   - Date/time
   - System affected
   - Brief description of incident

2. **Timeline**  
   - Chronological sequence of events
   - Key decision points
   - System responses

3. **Root Cause Analysis**  
   - Technical failures
   - Data anomalies
   - Configuration errors
   - Human factors

4. **Impact Assessment**  
   - Data integrity risks
   - Operational downtime
   - Reputational damage
   - Legal/ethical implications

5. **Corrective Actions**  
   - Immediate fixes
   - System updates
   - Process improvements
   - Training needs

6. **Lessons Learned**  
   - System vulnerabilities
   - Process gaps
   - Team coordination issues
   - Prevention strategies

7. **Prevention Plan**  
   - Monitoring enhancements
   - Redundancy improvements
   - Access controls
   - Regular audits

## Template Usage
Use this structure for post-mortem analysis after any AI system incident. Include technical details, affected systems, and recommended preventive measures.
