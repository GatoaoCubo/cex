## Purpose
A sandbox_config defines an isolated code execution environment for safe agent tool use: filesystem isolation, network access controls, process limits, and allowed system calls. It enables N05 to execute generated code, run tests, and validate builds in a contained environment without risking the host system.

## Pillar
P09 -- config

## Schema (key fields)
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| id | string | yes | Unique artifact identifier |
| kind | string | yes | Always `sandbox_config` |
| pillar | string | yes | Always `P09` |
| title | string | yes | Human-readable config name |
| version | semver | yes | Artifact version |
| quality | float\|null | yes | Peer-review score (null until reviewed) |
| engine | enum | yes | docker \| firecracker \| e2b \| wasm \| native |
| network | enum | yes | none \| restricted \| full |
| allowed_domains | list | no | Domains accessible when network=restricted |
| filesystem | enum | yes | readonly \| ephemeral \| persistent |
| max_cpu_cores | integer | no | CPU core limit |
| max_memory_mb | integer | yes | Memory ceiling in MB |
| max_disk_mb | integer | no | Ephemeral disk limit in MB |
| timeout_ms | integer | yes | Max execution time before kill |
| allowed_languages | list | no | Runtime languages available (python, node, bash) |

## When to use
- Executing agent-generated code in isolation before deploying to production
- Running test suites generated by N05 without host filesystem access
- Providing a safe execution layer for MCP tool calls that run arbitrary scripts
