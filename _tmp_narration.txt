Olá! Vou te explicar como funciona a nossa pipeline de pesquisa e CRM dentro do CEX. É um sistema bem engenhoso que usa inteligência artificial para construir uma base de contatos reais do zero.

O objetivo era simples: construir um CRM com mais de 500 contatos reais de negócios pet na região do ABC Paulista, usando apenas agentes de IA. Zero trabalho manual.

A pipeline funciona em 6 batches paralelos. Cada batch usa fontes diferentes de dados. Vou explicar cada um.

Batch A são os diretórios especializados. O agente N01 usa o servidor MCP do Firecrawl para fazer scraping de sites como Petlove Rede Credenciada, DogHero, PetAnjo, TeleListas e GuiaMais. Para cada uma das 7 cidades do ABC, ele extrai nome, telefone, endereço e categoria do negócio.

Batch B é o Google Maps. Aqui usamos o Brave Search como MCP server para buscar 9 categorias diferentes: pet shop, veterinária, banho e tosa, hotel pet, e mais. Multiplicado por 7 cidades, são 63 combinações de busca. O Google Maps é a fonte mais rica porque traz telefone, rating, reviews e coordenadas exatas.

Batch C é a descoberta via redes sociais. O agente varre Instagram e Facebook buscando hashtags como pet shop ABC, veterinário São Caetano, e assim por diante. Esse batch trouxe 53 contatos novos que não apareciam em nenhuma outra fonte.

Batch D são os marketplaces. O agente pesquisa vendedores pet no Mercado Livre, Shopee, iFood e Rappi. São negócios que já vendem online e estão prontos para parcerias digitais. Trouxe 12 contatos novos.

Batch E é reputação. O agente coleta reviews e ratings do Google e Reclame Aqui para enriquecer os contatos que já temos com dados de qualidade e satisfação do cliente.

E o Batch F é o CNAE Deep. Aqui o agente consulta bases governamentais como a Receita Federal via Casa dos Dados e CNPJá. Ele busca CNPJs ativos com códigos CNAE específicos do setor pet. Trouxe 55 CNPJs novos.

Tudo isso rodou em paralelo. O N07, que é o orquestrador, despachou 3 instâncias do N01 simultaneamente, cada uma trabalhando em batches diferentes. Quando os batches completaram, um script Python fez o merge, removeu duplicatas, e consolidou tudo num único arquivo JSON com 509 contatos reais.

Depois do merge, rodamos uma segunda fase chamada enriquecimento. Dividimos os contatos sem telefone em 3 batches regionais e despachamos 3 agentes N01 em paralelo novamente. Cada agente usou Brave Search e Firecrawl para encontrar telefones e WhatsApp que faltavam. Resultado: 241 contatos enriquecidos com 942 campos novos.

O stack técnico é o seguinte. O runtime é o Pi, que é um agente de código que roda Claude Opus com 1 milhão de tokens de contexto. Os servidores MCP fazem o trabalho pesado: Firecrawl para scraping de sites, Brave Search para buscas web, Fetch para download de páginas, e Markitdown para converter PDFs em texto.

O CRM armazena tudo em JSON localmente, com um dashboard HTML interativo que mostra os contatos num mapa usando Leaflet e OpenStreetMap. Cada contato tem nome, telefone, endereço, cidade, categoria, fonte de origem, e coordenadas geográficas.

O resultado final: 509 contatos reais, zero fabricados, cobrindo as 7 cidades do ABC Paulista. 60% com telefone, muitos com WhatsApp. Tudo construído por agentes de IA em aproximadamente 4 horas de processamento paralelo.

E o mais importante: toda essa pipeline virou template. Qualquer empresa pode rodar o mesmo processo mudando apenas as variáveis: região, indústria, fontes de dados. Os templates estão no repositório principal com variáveis mustache prontas para serem preenchidas.
