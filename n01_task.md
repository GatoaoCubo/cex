# CEX Nucleus Definition (N00-N07)

A CEX nucleus is a self-contained, fractal subsystem within the CEX architecture, operating with autonomy while maintaining systemic coherence. Each nucleus embodies a specific functional domain, governed by its own set of rules, tools, and workflows, yet interconnected through shared protocols and contextual awareness.

## Core Attributes

| Field | Description |
|-------|-------------|
| **nucleus_id** | Unique identifier (N00-N07) |
| **role** | Functional specialization (e.g., "Orchestrator", "Builder") |
| **pillars_owned** | 12-domain governance (mirrors N00's pillars) |
| **sin_lens** | Sin-aware UX filter for ethical compliance |
| **cli_binding** | Command-line interface entry point |
| **model_tier** | Model capability classification (Sonnet/Opus) |
| **boot_script** | Initialization script for nucleus activation |
| **agent_card_path** | Path to domain-specific agent configurations |
| **crew_templates_exposed** | Available templates for task composition |
| **domain_agents** | Specialized subagents for domain-specific tasks |

## Fractal Structure

Each nucleus operates as an autonomous fractal, containing:
- 13 subdirectories mirroring N00's 12 pillars + metadata
- 8F pipeline integration for artifact production
- Contextual memory systems for pattern recognition
- Autonomous dispatch capabilities via `_spawn/dispatch.sh`

The fractal architecture enables parallel processing while maintaining systemic coherence, allowing nuclei to specialize while contributing to the whole. This structure supports both granular task execution and holistic system evolution.
