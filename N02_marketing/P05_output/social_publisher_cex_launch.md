# CEX Public Launch: Social Publishing Strategy

## Campaign Premise

Most AI agent frameworks beg for attention. CEX earns it.

The positioning: **You've been building AI agents. CEX builds AI infrastructure.**
The proof: 293 kinds, 295 builders, 8 nuclei, 4 runtimes -- running on your repo, not ours.

This strategy does not describe CEX. It DEMONSTRATES it. Every post is a proof point.
Every thread is a live artifact. The content IS the product.

## Pipeline Overview

```
BRIEF (campaign goals) -> SELECT (platform-specific hook)
  -> GENERATE (copy variant A+B) -> OPTIMIZE (timing + hashtags)
  -> PUBLISH (platform API or manual) -> LOG -> NOTIFY -> ROTATE
```

## Platform Matrix

| Platform | Audience | Hook Vector | Content Type | Cadence | Best Time |
|----------|---------|-------------|-------------|---------|-----------|
| LinkedIn | CTOs, AI leads, founders | Authority + proof | Long-form threads | 3x/week | Tue-Thu 9am |
| X (Twitter) | Devs, indie hackers | Provocation + demos | Threads + clips | Daily | 8am + 5pm |
| GitHub | Open-source community | Proof + contribution | Stars, releases, discussions | At milestones | Any |
| Dev.to | Developer audience | Tutorial + walkthrough | Long articles | 2x/week | Mon/Wed |
| YouTube Shorts | Visual learners | Shock + awe demo | 60-sec terminal demos | 2x/week | Fri |

## Content Mix Strategy

| Category | % | What It Does |
|----------|---|-------------|
| Product proof (live demos, outputs) | 35% | Credibility -- show, don't tell |
| Provocation (contrast vs. competitors) | 25% | Stops the scroll |
| Education (8F, 12P, how-tos) | 20% | Builds authority over time |
| Community (contributor shoutouts, forks) | 15% | Social proof + network effects |
| Behind-the-scenes (build process) | 5% | Humanizes, builds trust |

### Dev.to -- The Tutorial Article

**Title:** "I gave each AI agent a deadly sin. Here's what happened."

**Format:** 2,000-word technical walkthrough
- Open with the sin architecture (hook -- unexpected concept)
- Walk through N02 Creative Lust in practice (concrete demo)
- Show the 8F pipeline on a real task
- Benchmarks vs. naive Claude prompting
- CTA: star the repo, read the docs

### GitHub -- The Credibility Layer

Content strategy for GitHub is NOT posts. It is ARTIFACTS:
- Comprehensive README that shows outputs, not just features
- GitHub Discussions: post weekly "build challenges" (community generates artifacts)
- Releases: tag each wave of builder additions with release notes that are themselves masterpieces of clarity
- Issues: respond within 24h, always with a working example

### YouTube Shorts -- The Shock Demo

**60-second script template:**
```
[0-3s] Visual: terminal blank screen
[3-8s] Type: "create marketing campaign for saas launch" (5 words)
[8-15s] Watch N02 run 8F pipeline in real time -- F1 through F8 labels animate

[45-55s] Cut to: the output files in the repo
[55-60s] Text overlay: "5 words -> professional campaign. CEX. Open source."

```

## Quality Gates

| Gate | Standard | Blocker? |
|------|---------|---------|
| CTA present | Every post has one clear next action | YES |
| Proof > description | Artifacts shown, not described | YES |
| Brand voice | Confident, not arrogant; technical but accessible | YES |
| Caption length | LinkedIn 200-1300 chars; X under 280 per tweet | YES |
| Hashtag count | LinkedIn <= 5; X <= 3 | NO |
| A/B variant | At least 2 versions per launch post | YES for paid |

## Anti-Patterns (Blocked)

- Feature lists without proof ("293 kinds" without showing ONE)
- Passive voice ("CEX can be used to...")
- Generic AI hype ("Revolutionary AI platform changes everything")
- Posting without CTA
- Identical copy across platforms (each platform = different seduction vector)
