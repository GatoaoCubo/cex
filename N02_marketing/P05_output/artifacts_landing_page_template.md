# Landing Page Templates — High-Converting Architectures

> A landing page is not a website. It has ONE job: make the reader do ONE thing.

## Universal Page Structure

```
┌─────────────────────────────────────┐
│  HERO — Stop. Read. Want.           │  Above the fold
│  (Headline + Subhead + CTA + Image) │
├─────────────────────────────────────┤
│  PROBLEM — "You know this pain"     │  Emotional hook
├─────────────────────────────────────┤
│  SOLUTION — "Here's the relief"     │  Product introduction
├─────────────────────────────────────┤
│  BENEFITS — "What changes for you"  │  3-5 benefit blocks
├─────────────────────────────────────┤
│  PROOF — "Others already did it"    │  Testimonials + logos
├─────────────────────────────────────┤
│  HOW — "Here's the path"            │  3-step process
├─────────────────────────────────────┤
│  OFFER — "Here's what you get"      │  Value stack
├─────────────────────────────────────┤
│  OBJECTIONS — "But what about..."   │  FAQ / guarantees
├─────────────────────────────────────┤
│  FINAL CTA — "Your move"            │  Urgency + action
└─────────────────────────────────────┘
```

## 1. Lead Capture Landing Page (TOFU)

**Goal**: Capture email in exchange for lead magnet
**Target conversion rate**: 25-45%

### Hero Section
```html
<section class="hero">
  <!-- HEADLINE: Specific outcome + audience -->
  <h1>[NUMBER] [OUTCOME_TYPE] to [SPECIFIC_RESULT] — Without [COMMON_PAIN]</h1>
  
  <!-- SUBHEADLINE: Expand on the promise -->
  <p>The exact [RESOURCE_TYPE] that helped [SOCIAL_PROOF_NUMBER] [AUDIENCE] 
     achieve [MEASURABLE_RESULT] in [TIMEFRAME].</p>
  
  <!-- VISUAL: Lead magnet mockup or hero image -->
  <img src="[LEAD_MAGNET_MOCKUP]" alt="[DESCRIPTIVE_ALT_TEXT]" />
  
  <!-- CTA: First-person, benefit-specific -->
  <form>
    <input type="email" placeholder="Your best email" required />
    <button type="submit">Yes, Send Me the [RESOURCE_TYPE]</button>
    <p class="micro-copy">Free. No spam. Unsubscribe anytime.</p>
  </form>
</section>
```

### What's Inside Section
```
Inside this [RESOURCE_TYPE], you'll discover:

✓ [SPECIFIC_BENEFIT_1] — [one-line expansion]
✓ [SPECIFIC_BENEFIT_2] — [one-line expansion]
✓ [SPECIFIC_BENEFIT_3] — [one-line expansion]
✓ [SPECIFIC_BENEFIT_4] — [one-line expansion]
✓ BONUS: [UNEXPECTED_VALUE] — [why it matters]
```

### Social Proof Strip
```
"[SHORT_TESTIMONIAL_QUOTE]" — [NAME], [ROLE/COMPANY]

[METRIC_1]: [NUMBER]+ downloads
[METRIC_2]: [RATING]★ average rating
[METRIC_3]: Featured in [PUBLICATION]
```

### Final CTA (repeat)
```
Don't let another [TIME_UNIT] go by struggling with [PROBLEM].

[SAME_CTA_BUTTON_AS_HERO]

You're protected by our [GUARANTEE]. No risk. All upside.
```

## 2. Sales Page (BOFU)

**Goal**: Convert warm leads to buyers
**Target conversion rate**: 2-8%

### Hero Section
```
<!-- HEADLINE: Transformation statement -->
<h1>From [CURRENT_STATE] to [DESIRED_STATE] — in [TIMEFRAME]</h1>

<!-- SUBHEADLINE: How + social proof -->
<p>Join [NUMBER]+ [AUDIENCE] who used [PRODUCT_NAME] to [SPECIFIC_RESULT].</p>

<!-- DUAL CTA -->
<button class="primary">Get [PRODUCT_NAME] Now — $[PRICE]</button>
<a class="secondary">Watch the 2-minute demo ▶</a>
```

### Problem Section (PAS Framework)
```
## Still [STRUGGLING_WITH_PROBLEM]?

You've tried [COMMON_SOLUTION_1]. And [COMMON_SOLUTION_2]. Maybe even [COMMON_SOLUTION_3].

But here's what nobody tells you: [HIDDEN_REASON_THEY_FAIL].

The result? [PAINFUL_CONSEQUENCE_1]. [PAINFUL_CONSEQUENCE_2]. 
And the nagging feeling that [EMOTIONAL_PAIN].

What if there was a better way?
```

### Solution Section
```
## Introducing [PRODUCT_NAME]

[ONE_SENTENCE_DESCRIPTION — what it IS]

Unlike [ALTERNATIVES], [PRODUCT_NAME] [KEY_DIFFERENTIATOR] so you can 
[PRIMARY_BENEFIT] without [PRIMARY_OBJECTION].

[PRODUCT_SCREENSHOT_OR_VIDEO]
```

### Benefit Stack (FAB Framework)
```
### What [PRODUCT_NAME] Does For You

┌──────────────────────────────────────┐
│ [FEATURE_1]                          │
│ → [ADVANTAGE]: [what it does]        │
│ → [BENEFIT]: [what it means for you] │
├──────────────────────────────────────┤
│ [FEATURE_2]                          │
│ → [ADVANTAGE]: [what it does]        │
│ → [BENEFIT]: [what it means for you] │
├──────────────────────────────────────┤
│ [FEATURE_3]                          │
│ → [ADVANTAGE]: [what it does]        │
│ → [BENEFIT]: [what it means for you] │
└──────────────────────────────────────┘
```

### Social Proof Section
```
## Don't Take My Word For It

[TESTIMONIAL_1 — specific result + before/after]
"Before [PRODUCT], I was [BEFORE_STATE]. Now [AFTER_STATE]. 
The best part? [SPECIFIC_METRIC_IMPROVEMENT]."
— [NAME], [TITLE] at [COMPANY]

[TESTIMONIAL_2 — emotional transformation]
[TESTIMONIAL_3 — skeptic-turned-believer]

LOGOS: [COMPANY_1] · [COMPANY_2] · [COMPANY_3] · [COMPANY_4]
```

### How It Works (3-Step)
```
## Get Started in 3 Simple Steps

1️⃣ [ACTION_VERB] — [what they do] (takes [TIME])
2️⃣ [ACTION_VERB] — [what happens next] (takes [TIME])
3️⃣ [ACTION_VERB] — [the outcome they experience]

That's it. No [COMMON_COMPLEXITY]. No [COMMON_HASSLE].
```

### Offer Stack
```
## Here's Everything You Get

✅ [CORE_PRODUCT] ........................ Value: $[AMOUNT]
✅ [BONUS_1] ............................. Value: $[AMOUNT]
✅ [BONUS_2] ............................. Value: $[AMOUNT]
✅ [BONUS_3] ............................. Value: $[AMOUNT]

Total Value: $[SUM]
━━━━━━━━━━━━━━━━━━━━━━━
Your Price Today: $[ACTUAL_PRICE]

[CTA_BUTTON: Get Everything Now — $[PRICE]]
```

### Objection Handler (FAQ)
```
## Common Questions

Q: [BIGGEST_OBJECTION — usually price or time]
A: [REFRAME_AS_INVESTMENT + comparison to cost of NOT buying]

Q: [SECOND_OBJECTION — usually "will it work for me?"]
A: [SEGMENT_SPECIFIC_PROOF + guarantee mention]

Q: [THIRD_OBJECTION — usually technical/logistical]
A: [SIMPLE_ANSWER + support mention]

Q: What if it doesn't work for me?
A: [GUARANTEE_DETAILS — risk reversal is THE conversion lever]
```

### Final CTA Section
```
## [TRANSFORMATION_HEADLINE]

You have two choices right now:

Option A: Close this page and go back to [CURRENT_PAINFUL_STATE].

Option B: [TAKE_ACTION] and start [EXPERIENCING_DESIRED_STATE] today.

[LARGE_CTA_BUTTON]

🔒 [GUARANTEE_BADGE] · [PAYMENT_SECURITY] · [SUPPORT_PROMISE]

P.S. [URGENCY_OR_FINAL_BENEFIT — restate the strongest argument in 2 sentences]
```

## 3. Webinar/Event Registration Page (MOFU)

**Goal**: Register for live event
**Target conversion rate**: 20-40%

### Structure
```
HERO:
  headline: "[EVENT_TYPE]: How to [SPECIFIC_OUTCOME] — Live on [DATE]"
  subhead: "Join [SPEAKER] for [DURATION] of [VALUE_DESCRIPTION]"
  cta: "Save My Spot"
  urgency: "[NUMBER] spots remaining"

WHAT_YOU_LEARN:
  bullet_1: "[TAKEAWAY_1]"
  bullet_2: "[TAKEAWAY_2]"
  bullet_3: "[TAKEAWAY_3]"
  bonus: "Plus live Q&A — get your specific questions answered"

SPEAKER_BIO:
  photo: "[HEADSHOT]"
  name: "[NAME]"
  credentials: "[RELEVANT_AUTHORITY_MARKERS]"
  social_proof: "[SPEAKING_HISTORY or RESULTS]"

TESTIMONIALS:
  past_attendee_1: "[QUOTE about value received]"
  past_attendee_2: "[QUOTE about specific takeaway]"

LOGISTICS:
  date: "[DATE + TIMEZONE]"
  duration: "[TIME]"
  platform: "[WHERE]"
  replay: "Can't make it live? Register anyway — we'll send the replay."
  
FINAL_CTA:
  headline: "Don't miss this."
  button: "Reserve My Seat — Free"
  scarcity: "Limited to [NUMBER] live attendees"
```

## 4. Coming Soon / Waitlist Page (Pre-Launch)

**Goal**: Build anticipation + capture early interest
**Target conversion rate**: 30-60%

### Structure
```
HERO:
  headline: "Something [ADJECTIVE] is coming to [INDUSTRY/NICHE]"
  subhead: "[ONE_SENTENCE_TEASER — enough to intrigue, not enough to satisfy]"
  cta: "Join the Waitlist — Be First to Know"
  counter: "[NUMBER] people already waiting"

TEASER_DETAILS:
  reveal_1: "[PARTIAL_DETAIL — what it does]"
  reveal_2: "[PARTIAL_DETAIL — who it's for]"
  mystery: "Full details drop [DATE]. Waitlist members get [EARLY_ACCESS/DISCOUNT]."

SOCIAL_PROOF:
  endorsement: "[INDUSTRY_AUTHORITY] says: '[QUOTE]'"
  
COUNTDOWN:
  target_date: "[LAUNCH_DATE]"
  display: "Days : Hours : Minutes : Seconds"
```

## Conversion Optimization Checklist

```yaml
above_the_fold:
  - [ ] Headline visible without scrolling
  - [ ] CTA button visible without scrolling
  - [ ] Value proposition clear in 5 seconds
  - [ ] No navigation menu (single-purpose page)

copy_quality:
  - [ ] Headline passes 4U test (Useful, Urgent, Unique, Ultra-specific)
  - [ ] All CTAs use first-person ("Get MY guide" not "Get YOUR guide")
  - [ ] Benefits > Features (3:1 ratio minimum)
  - [ ] Social proof within first 2 scrolls
  - [ ] Guarantee/risk reversal present

technical:
  - [ ] Page loads under 3 seconds
  - [ ] Mobile-responsive (test on 3 devices)
  - [ ] Form has maximum 3 fields (email minimum)
  - [ ] Thank-you page configured (not just "thanks")
  - [ ] Tracking pixels installed (Meta, Google, UTMs)
  - [ ] HTTPS (no mixed content warnings)

design:
  - [ ] Single visual hierarchy path (F or Z pattern)
  - [ ] Contrast ratio >= 4.5:1 on all text
  - [ ] CTA button color contrasts with page (no blending)
  - [ ] White space between sections (breathing room)
  - [ ] Consistent typography (max 2 font families)

psychology:
  - [ ] Specific numbers > vague claims
  - [ ] "You/Your" > "We/Our" (2:1 ratio minimum)
  - [ ] Urgency element present (deadline, scarcity, or FOMO)
  - [ ] Risk reversal present (guarantee, free trial, refund)
  - [ ] Social proof present (testimonials, logos, metrics)
```

## A/B Testing Priority for Landing Pages

| Priority | Element | Expected Impact |
|----------|---------|-----------------|
| 1 | Headline (pain vs benefit vs curiosity) | 20-40% conversion lift |
| 2 | CTA text (action vs benefit) | 10-25% conversion lift |
| 3 | Hero image (product vs people vs abstract) | 5-15% conversion lift |
| 4 | Form length (1 field vs 3 fields) | 10-30% conversion lift |
| 5 | Social proof placement (above vs below fold) | 5-10% conversion lift |
| 6 | Page length (short vs long) | Depends on awareness level |
