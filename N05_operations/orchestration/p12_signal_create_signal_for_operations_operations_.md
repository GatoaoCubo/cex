# Signal Schema

## Field Definitions
- **satellite**: (string, required) The lowercase slug of the emitting agent or satellite.
- **status**: (enum, required) Status of the event; valid values: `complete`, `error`, `progress`.
- **quality_score**: (null, required) The quality score of the event outcome; must be `null` as self-scoring is not allowed.
- **timestamp**: (string, required) ISO 8601 timestamp of when the event occurred.

## Optional Fields
- **task**: (string, optional) A short label describing the specific task related to the event.
- **artifacts**: (array of strings, optional) Paths to any artifacts generated by this event.
- **artifacts_count**: (integer, optional) Number of artifacts produced.
- **commit_hash**: (string, optional) Git commit hash related to the event.
- **error_code**: (string, optional) Short code identifying the error, if applicable.
- **message**: (string, optional) Brief human-readable message about the event.
- **progress_pct**: (integer, optional) Percent completion of the event if status is `progress`.

# Status Vocabulary

- **complete**: Indicates the event has successfully concluded, and outputs are ready to be consumed.
- **error**: Denotes that an event error occurred, requiring corrective action or attention.
- **progress**: Shows an event is ongoing with current progress known; should include `progress_pct`.

# Consumer Contract

- **MUST handle**: `satellite`, `status`, `quality_score`, `timestamp`.
- **MAY ignore**: `task`, `artifacts`, `artifacts_count`, `commit_hash`, `error_code`, `message`, `progress_pct` if not applicable to consumer context.
