# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |

## Reporting a Vulnerability

Use **GitHub's private vulnerability reporting** to disclose security issues:

1. Go to the **Security** tab of this repository
2. Click **Report a vulnerability**
3. Fill in the advisory form with:
   - a clear description of the issue
   - steps to reproduce it
   - the affected version, branch, or commit if known
   - any proof-of-concept or logs needed to validate the report

This keeps the report private between you and the maintainers until a fix is ready.

Do not open a public GitHub issue for suspected vulnerabilities.

## Response Process

- We will acknowledge receipt of a report as soon as practical.
- We will investigate the issue and determine severity and impact.
- We may ask for more detail or a reproduction case.
- We will coordinate a fix and disclosure plan with the reporter when appropriate.

## Disclosure Policy

This project follows a 90-day coordinated disclosure window. Our goal is to investigate, remediate, and publish a fix or mitigation within 90 days of the initial report when feasible.

If a fix takes longer, maintainers may extend the window when the risk of immediate public disclosure is higher than the risk of a delayed advisory.

## Scope

This policy applies to vulnerabilities in:

- source code in this repository
- build and dispatch scripts
- default configuration and generated artifacts

Third-party dependencies should also be reported to the upstream maintainer when the issue originates outside this repository.
