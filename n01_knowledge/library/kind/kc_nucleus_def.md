# Nucleus Definition (N00-N07)

A CEX nucleus is a self-contained, fractal subsystem within the CEX architecture, operating under these formal constraints:

## Core Attributes
- **nucleus_id**: Unique identifier (N00-N07)
- **role**: Functional purpose (e.g., "Orchestrator", "Builder")
- **pillars_owned**: 12 pillars managed (e.g., N01-Intelligence)
- **sin_lens**: Sin framework compliance (e.g., "N01-Intelligence")
- **cli_binding**: Command-line interface binding (e.g., `n03-builder`)
- **model_tier**: Model tier classification (e.g., "Opus", "Sonnet")
- **boot_script**: Boot script path (e.g., `boot/n03.ps1`)
- **agent_card_path**: Agent configuration path (e.g., `agents/n03.json`)
- **crew_templates_exposed**: Available template paths (e.g., `templates/n03/*.md`)
- **domain_agents**: Specialized agents (e.g., `n03-builder`, `n04-librarian`)

## Fractal Structure
Each nucleus mirrors N00's 12 pillars, creating a recursive architecture where:
1. N00 (Genesis) defines the root fractal
2. N01-N07 (Nuclei) each own 12 pillars
3. ISOs (13 per kind) implement pillar-specific functions
4. Sub-agents (125 total) execute domain-specific tasks

This structure enables autonomous operation while maintaining systemic coherence across all nuclei.
