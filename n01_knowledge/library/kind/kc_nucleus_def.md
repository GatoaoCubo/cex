# Nucleus Definition (Knowledge Contract)

## Fields
- **nucleus_id**: Unique identifier for this nucleus (N00-N07)
- **role**: Core function within the CEX architecture (e.g., orchestrator, builder)
- **pillars_owned**: List of 12 pillars this nucleus manages
- **sin_lens**: Sin-aware perspective for ethical decision-making
- **cli_binding**: Command-line interface binding for nucleus interaction
- **model_tier**: Model tier classification (Sonnet/Opus)
- **boot_script**: PowerShell script for nucleus initialization
- **agent_card_path**: Path to associated agent configuration
- **crew_templates_exposed**: Templates available for crew composition
- **domain_agents**: List of specialized agents within this nucleus

## Description
This knowledge card defines the structural and functional characteristics of a CEX nucleus. It establishes the fractal architecture by explicitly mapping each nucleus to its 12 pillars, sin-aware constraints, and operational interfaces. The definition enables precise system composition and ensures alignment with the CEX framework's architectural principles.

## Key Relationships
- **N07 Orchestrator**: Manages dispatch to other nuclei
- **N03 Builder**: Executes creation tasks with high-tier models
- **P01 Knowledge**: Provides foundational definitions for all nuclei
